## CONTEXT

The current Skills section displays two columns of skills with percentage bars and numbers (e.g. "Strategic Brand Management — 95%"). This is being replaced with a **Capability Cards** layout — a 2×3 grid of glassmorphism cards, each showing a skill name, a one-line proof statement, and category tags. No numbers, no bars.

**Design system rules that apply here:**
- Font: Space Grotesk only, no exceptions
- No rounded corners (`border-radius: 0` everywhere)
- No light backgrounds — all dark (`#03000A` base)
- Card style: glassmorphism panels (`rgba(255,255,255,0.04)` background, `rgba(255,255,255,0.07)` border)
- Hover: `rgba(255,255,255,0.07)` background + `rgba(163,120,255,0.3)` border + `0 0 40px rgba(123,94,167,0.15)` box-shadow
- Animations: Framer Motion `{ opacity: 0, y: 24 }` → `{ opacity: 1, y: 0 }`, duration 0.7s, ease "easeOut"
- Section label prefix: `//`
- Do NOT touch routing, backend, database, admin system, or any other component

## STEP 4 — Mobile responsiveness

The grid is `repeat(2, 1fr)` on desktop. Add a responsive breakpoint so it stacks to a single column on mobile. If the project uses Tailwind for media queries, the inline grid style can be handled with a conditional or a className. 

If the existing Skills component used a className-based approach for responsive grids, match that pattern. The simplest inline approach is to add this to the grid container:

```tsx
// Add this state at the top of the component:
const [isMobile, setIsMobile] = React.useState(false);

React.useEffect(() => {
  const check = () => setIsMobile(window.innerWidth < 768);
  check();
  window.addEventListener("resize", check);
  return () => window.removeEventListener("resize", check);
}, []);

// Then use it on the grid:
gridTemplateColumns: isMobile ? "1fr" : "repeat(2, 1fr)",
```

Make sure to add `import React from "react"` at the top if it isn't already there.
